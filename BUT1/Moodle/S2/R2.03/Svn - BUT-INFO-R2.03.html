    Svn

    Nicolas Le Sommer, Jean-François Kamp et Régis Fleurquin

    1. Introduction#

    Subversion (SVN) est un système de gestion de versions. SVN reprend les principaux concepts du système de gestion de version CVS (dépôt centralisé et unique). Il comble certains manques de CVS, comme la possibilité de renommer ou de déplacer des fichiers.

    Contrairement à CVS, dans SVN les numéros de révision sont globaux et associés à l’ensemble du dépôt et plus à un fichier particulier. Chaque patch à un numéro de révision, quels que soient les fichiers touchés. On peut identifier simplement une version particulière d’un logiciel grâce à son numéro de révision.

    2. Quelques outils#

    • Outils en ligne de commandes sous Unix: subversion (svn), diff, patch
    • Outils graphiques: RapidSVN (sous Unix et Windows), TortoiseSVN (sous Windows)
    • Outils intégrés à des IDE: Subclipse pour Eclipse
    • Outils graphiques de calcul de différences: meld sous Unix

    3. Travail à réaliser#

    Pour ce travail, vous rédigerez un document indiquant quelles commandes vous avez exécutées pour les questions suivantes, et quelles réponses vous avez obtenues.

    1. Créez, à l’aide de la commande svnadmin, un dépôt central SVN myrepo sur votre machine dans le répertoire /tmp/.

    2. Créez l’arborescence de répertoires suivante et importer celle-ci dans votre dépôt. 

      myapp/
├── doc
└── src

    3. Supprimez le répertoire myapp et obtenez une version de celui-ci depuis le dépôt myrepo.

    4. Ajoutez un fichier README.md contenant vos noms et prénoms dans le répertoire myapp.

    5. Vous considérerez pour la suite que le repertoire myapp est celui d’un utilisateur A. Pour cela, créez dans le répertoire /tmp un répetoire userA et déplacez le répetoire myapp dans celui-ci. Créez un répetoire userB dans le répertoire /tmp et créez une nouvelle copie locale du dépôt myrepo dans ce répertoire userB. Dans la suite, ces deux répertoires userA et userB représenteront les répetoires personnels de developpeurs A et B.

    6. Suivez le scénario suivant et donnez les commandes exécutées.

      • Le développeur A ajoute un fichier Java contenant une classe dans le répertoire src. Il ce fichier au dépôt.

      • Le développeur B recupère le fichier Java déposé par A.

      • B modifie le fichier Java déposé par A, et propage sa modification sur le dépôt central.

      • A modifie le fichier Java, et propage sa modification sur le dépôt central.

    7. Décrivez la démarche que vous avez suivie pour résoudre le conflit.

    8. Reprenez le scénario précédent, mais cette fois, posez un verrou pour éviter le conflit. Donnez les commandes que vous avez exécutées pour les développeurs A et B.

    Vous pourrez utiliser la commande svn status pour connaître l’état de vos fichiers dans votre copie locale.
